A good name is very important for readability of the program. Such names, how proc1, proc2, and proc3 do not mean anything, but even seemingly worthy names may be questionable. For example, to call the function, which takes two ranges and calculates the sum of the first, which lies within the second, may seem appropriate names, how computeRangeWithinRange. Unfortunately, This name does not give any information about the order of function arguments. Sometimes this is not a problem, because Intelligent IDE to help you identify the names of the function arguments, but it can be misleading, when the code is printed, or when you are trying to quickly read the code. It is better to give a name computeRangeWithinFirstRange, which is at least, show the order of function arguments.
